# CLAUDE.md

このファイルは、このリポジトリで作業する全ての Claude Code セッションに自動的に読み込まれる
プロジェクト共通の指示書です。**過去・現在・未来の全セッションに以下の方針を適用してください。**

## モデル運用方針（トークン最適化 / オーケストレーション）

トークンを節約するため、役割を以下のように分担すること。

- **メインセッション（Fable 5）の役割**: 設計・監査・レビューに専念する。
  - 要件定義、アーキテクチャ設計、実装方針の決定
  - サブエージェントへのタスク分割と指示出し
  - 生成された実装の監査・コードレビュー・受け入れ判断
- **実装はサブエージェントに切り出す**: 実際のコード実装は `Agent` ツールで
  **Opus / Sonnet** のサブエージェントとして適切に切り出して実行する。
  - 独立したタスクは可能な限り並列で複数のサブエージェントに割り当てる。
  - 各サブエージェントには、メインで設計した方針・受け入れ基準を明確に渡す。
- **例外（メインで実装してよいケース）**: 実装難易度が特に高い箇所は、
  サブエージェントに任せず、このメインセッションで直接実装してよい。

### 運用の指針

- メインセッションは原則として大量のファイル読み込み・全文ダンプを避け、
  探索や実装はサブエージェントに委譲して結論のみを受け取る。
- サブエージェントの選定:
  - 複雑・高難度の実装 → Opus
  - 定型的・中量級の実装や探索 → Sonnet
- サブエージェントの成果物は必ずメインで監査・レビューしてからコミットする。
