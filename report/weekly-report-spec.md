# うにの占いの館 — 週次レポート仕様

毎週日曜 12:00（日本時間）に、直近7日間の評価データを集計して
martinniche@gmail.com 宛のレポートを作成する。

## データソース

アプリの「さつまいも評価」（3段階）送信時に、Netlify Forms `uni-hyoka` に以下が記録される。
Netlify の Form notifications により、送信ごとの通知メールが 5newold111@gmail.com に届く。
週次集計はこの通知メールを Gmail 検索（from:formresponses@netlify.com など、件名 uni-hyoka）で収集して行う。

| フィールド | 内容 |
|---|---|
| rating | 1=少し / 2=普通 / 3=たくさん |
| ratingLabel | 少し・普通・たくさん |
| theme | 相談ジャンル（総合運/恋愛運/仕事運/金運） |
| spread | 占い方（ワンオラクル/スリーカード/状況スプレッド） |
| topic | 相談トピック13分類（復縁/片思い/結婚/別れ・離婚/転職/職場の人間関係/新しい挑戦/お金/友人関係/家族/心と体/進路・学び/住まい）または「未分類」「相談なし」 |
| concern | 相談本文（200字まで） |
| cards | 引いた札と正逆（例: 運命の輪(正)、戦車(正)、隠者(正)） |
| upright / total | 正位置の枚数 / 引いた枚数 |
| ts / tz | 送信日時(ISO・UTC) / 利用者のタイムゾーン |

## レポートの10項目

1. **総鑑定数**（週合計の評価送信数、前週比）
2. **相談ジャンル別の件数分布**（総合/恋愛/仕事/金運の割合）
3. **相談トピック別の件数ランキング**（13分類+未分類。どんな悩みが多いか）
4. **実際の相談文の代表例**（多数派トピックから数件、個人特定情報はマスク）
5. **評価の全体分布**（少し/普通/たくさんの件数と平均スコア 1.00〜3.00）
6. **ジャンル別の平均評価**（評価が高いジャンル・低いジャンル）
7. **トピック別の平均評価**（満足度の高い悩み・低い悩みの特定 → 鑑定文の改善対象）
8. **スプレッド別の利用数と平均評価**（どの占い方が好まれ、満足されているか）
9. **曜日・時間帯の傾向**（tsをJSTに変換し、利用が集中する曜日・時間帯）
10. **札の正位置率と評価の相関**（正位置が多い結果ほど高評価か。逆位置時の
    フォロー文の改善判断に使う）

末尾に「今週の所見と改善提案」を2〜3行で添える。
