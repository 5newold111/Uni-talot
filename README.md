# シーズーうにの占いの館

深い夜のインクとアンティークな真鍮金に包まれた、本物のシーズー犬「**うに**」がいるタロット占いの館。
大アルカナ22枚で、あなたの今を静かに読み解きます。ブラウザだけで動作・インストール不要・完全オフライン。

> 「やっほー、来てくれて嬉しいワン。うにと一緒に視てくれる？……あ、いま さつまいも食べてたの。えへへ」― 占い師うに（女の子・食いしん坊・ちょっと甘えん坊）

## 遊び方

1. `index.html` をブラウザで開く（うにの館へ入る）
2. 占いたい **テーマ**（総合運・恋愛運・仕事運・金運）を選ぶ
3. **占い方（スプレッド）** を選ぶ
4. 願いを念じながら、裏向きの札を引く
5. 占い師うにが、引いた札を読み解いて鑑定してくれる

## デザイン

実在の占い・タロットサイトの手法を参考に、"AIっぽさ"を避けた editorial なビジュアルに設計しています（絵文字は不使用）。

- **Golden Thread Tarot / Labyrinthos** … 暗いマット地に金の細線（フィリグリー）と幾何学的な意匠
- **アストロロジー系タロットクロス** … 派手な金ではなくアンティークな真鍮金で上品に、星と月の縁取り
- **しいたけ占い（VOGUE）** … 明朝体の editorial な見出し、余白、テーマ別カラーコード
- **アール・ヌーヴォー（ミュシャ系）タロット** … アーチ型の装飾枠と飾り罫（フィリグリー）
- **ライダー＝ウェイト版** … ローマ数字のカード番号、ギルト縁、正逆位置の作法

素材（星空・アーチ枠・カード背面・テーマ図像・水晶・飾り罫）はすべて Canvas / SVG で生成し、うにの肖像はアーチ枠に納めています。

## 特徴

- 占い師「うに」の一貫した世界観 — 本物のうにの写真を肖像に、語りかけ・見立て・署名まで（女の子・さつまいも好き・ちょっと甘えん坊）
- 大アルカナ22枚を完全収録。各カードに「本質」＋テーマ別の正位置・逆位置の解釈
- 深い鑑定 — 各カードは「読み解き」＋「具体的アドバイス」の二段構成
- 引いたカードを一つの物語として関連づける総合鑑定文（流れ・全体トーン・締めの言葉）
- 3種類のスプレッド（ワンオラクル／スリーカード／状況スプレッド）
- シャッフルとランダムな正逆位置で毎回違う結果
- 水晶玉を覗き込む占い演出、星空アニメーション
- スマホ対応のレスポンシブ、`prefers-reduced-motion` 配慮
- **PWA対応** … スマホのホーム画面に追加でき、オフラインでも起動（`manifest.webmanifest` ＋ Service Worker）
- **SNSシェア最適化** … OGP / Twitterカードで、リンク共有時にうにの写真つきプレビューを表示
- **結果シェアボタン** … 占い結果を Web Share API（対応外はリンクコピー）で友だちに共有

## Webアプリとして公開する（多くの人に届ける）

このリポジトリは静的サイトなので、次のいずれかで無料公開できます。どれも独自URLで配布可能です。

### 方法1：GitHub Pages（このリポジトリを公開する場合）
1. リポジトリを **Public** にする（Settings → General → Danger Zone → Change visibility）
2. Settings → **Pages** → Build and deployment → Source を **「GitHub Actions」** に設定
3. 以後、`main` への push で `.github/workflows/deploy.yml` が自動デプロイ
4. 公開URL： `https://5newold111.github.io/furniture-3D-modeling-/`

### 方法2：Cloudflare Pages / Netlify（リポジトリを非公開のまま公開したい場合）
- **Netlify**： [app.netlify.com/drop](https://app.netlify.com/drop) にプロジェクトのフォルダをドラッグ＆ドロップするだけで即時公開（アカウント連携なしでも可）。
- **Cloudflare Pages / Vercel**： GitHub 連携で非公開リポジトリからもデプロイ可能。ビルド設定なし（フレームワーク: None、出力: ルート）。
- 公開後は `index.html` の OGP メタ（`og:url` / `og:image`）と `manifest` の URL を、実際のドメインに合わせて更新してください。

### 公開後に伸ばすための施策（提案）
- **独自ドメイン**（例：`uni-tarot.app`）を割り当てると信頼感と記憶性が上がる
- **アクセス解析**（Cloudflare Web Analytics や Plausible などプライバシー配慮型）で流入を把握
- **日替わりの一枚**や**結果カードの画像保存**機能を追加して再訪・シェアを促進
- **X / Instagram**（本家 @shih_tzu_uni）と連携し、うにの世界観で導線をつくる

## ファイル構成

アプリ本体は `index.html` の **1ファイル** で完結（CSS・JavaScript・カードデータ・うにの写真をすべて内包）。
ローカルで遊ぶだけならこの1ファイルを開くだけでOK。以下はWebアプリ（PWA）として公開するための付随ファイルです。

| ファイル | 役割 |
|---|---|
| `index.html` | アプリ本体（自己完結） |
| `manifest.webmanifest` | PWA設定（アプリ名・アイコン・表示） |
| `sw.js` | Service Worker（オフライン対応・キャッシュ） |
| `icon-192.png` / `icon-512.png` | アプリアイコン（うにの写真から生成） |
| `apple-touch-icon.png` / `favicon-32.png` | iOS・ブラウザ用アイコン |
| `og-image.png` | SNS共有プレビュー画像（1200×630） |
| `uni.jpg` | うにの元写真（素材） |
| `.github/workflows/deploy.yml` | GitHub Pages 自動デプロイ |

## おまけアプリ：おそうじ当番表（`toban/`）

家庭内の掃除当番を自動でローテーションして管理できるアプリです。`toban/index.html` の1ファイルで完結し、公開すると `/toban/` で使えます（ローカルではファイルを開くだけ）。

- 家族メンバーを登録（並び順＝当番の順番、↑↓で入れ替え可）
- 掃除場所を「毎日」「週1回」の頻度つきで登録（よくある場所はワンタップ追加）
- 日付から当番を自動で公平に割り振り。「⇄」ボタンでその日だけ交代もできる
- 今日・今週の当番の完了チェックと、月間・累計の実績ランキング
- データは端末のブラウザに保存（書き出し／取り込みでバックアップ・引っ越し可）
- **Google連携（任意）** … 家族それぞれのGoogleアカウントでログインして、同じ当番表をリアルタイム共有。自分の当番だけの表示（棲み分け）や、Googleカレンダー経由の通知ができる（下記）

### 家族のGoogleアカウントと連携する（任意・無料）

連携しなくても1台の端末でそのまま使えます。家族それぞれのスマホで同じ当番表を使いたい場合だけ、**最初のひとりが一度だけ** 以下の準備をしてください（無料のFirebaseを利用。2人目以降は招待リンクを開くだけ）。

1. **Firebaseプロジェクトを作る**
   [console.firebase.google.com](https://console.firebase.google.com/) → 「プロジェクトを追加」（名前は自由、Googleアナリティクスは不要）
2. **Webアプリを追加して設定をコピー**
   プロジェクトの概要 → `</>`（ウェブ）アイコン → アプリ登録 → 表示される `const firebaseConfig = { ... }` を丸ごとコピー
3. **Googleログインを有効化**
   構築 → Authentication → 始める → Sign-in method → **Google** を有効化
   さらに Authentication → Settings → **承認済みドメイン** に公開先ドメイン（例：`○○○.netlify.app`）を追加
4. **Firestoreを作る**
   構築 → Firestore Database → データベースを作成（本番モード・ロケーションは `asia-northeast1` など）→ 「ルール」タブを以下に書き換えて公開：
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /families/{familyId} {
         allow read, write: if request.auth != null;
       }
     }
   }
   ```
5. **（通知を使う場合）Google Calendar API を有効化**
   [console.cloud.google.com](https://console.cloud.google.com/) で同じプロジェクトを選び、「APIとサービス」→ ライブラリ → **Google Calendar API** を有効化。
   「OAuth同意画面」→ 対象（Audience）→ **テストユーザー** に家族全員のGmailアドレスを追加（公開ステータスは「テスト」のままでOK）
6. **アプリで設定**
   公開URLで当番表を開く → 設定タブ → 「家族で共有」→ firebaseConfig を貼り付けて保存 → Googleでログイン → 「家族グループを作る」→ **招待リンクをコピー** して家族に送る

家族は届いた**招待リンクを開いてGoogleでログインするだけ**。あとは各自が「あなたはどのメンバー？」で自分を選べば、

- 当番表がリアルタイムで全員に同期される
- 「今日」タブで自分の当番が強調表示され、「自分だけ」フィルタで棲み分けできる
- 設定タブの「Googleカレンダーで通知」から、自分の当番（14日分）を自分のカレンダーに登録でき、指定時刻にスマホへ通知が届く（週1回の当番は土曜日にまとめて通知）

注意：共有機能は公開URL（https）で開いたときだけ使えます（`file://` では不可）。招待リンクには接続情報が含まれるので家族以外に教えないでください。ログインせずに使う端末は、従来どおりローカル保存のまま動きます。
