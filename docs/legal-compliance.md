# 著作権・法的配慮ガイド

本システムは「既存の楽曲を解析して似た曲を作る」ものでは**ありません**。
著作権・著作隣接権を侵害しないために、以下の原則を厳守する設計になっています。

## 基本原則

### 1. 既存楽曲そのものを解析・抽出しない

- 特定の楽曲の **音源・メロディ・歌詞** を取り込み、特徴を抽出して再生産する処理は
  実装しません。これは翻案権・複製権の侵害リスクが高いためです。
- 代わりに、**公開されている統計的・抽象的なトレンド情報**のみを扱います。
  - 例: 「最近はミッドテンポ（90–110 BPM）の Lo-Fi ポップが伸びている」
  - 例: 「夏季はアップリフティングなムードの需要が高い」
  - これらは**アイデア・事実・傾向**であり、著作権の保護対象ではありません
    （アイデア・表現二分論）。

### 2. オリジナルな表現を生成する

- 歌詞・メロディ・コード進行は、トレンドの**抽象的特徴**を入力に、Gemini と SUNO が
  **ゼロからオリジナル**に生成します。特定楽曲の表現を写し取ることはしません。
- タイトル・歌詞に既存楽曲やアーティスト名、商標を流用しないよう、生成プロンプトに
  ガードレールを入れています（`trend_analysis.py` のシステムプロンプト参照）。

### 3. 生成物の権利と各サービスの規約

- **SUNO**: 有料プランで生成した楽曲の商用利用可否は SUNO の利用規約に従います。
  利用前に最新の規約・商用利用条件を必ず確認してください。
- **DistroKid**: AI 生成楽曲の配信可否・表記要件は各ストア/ディストリビューターの
  ポリシーに従います（Spotify 等は AI 生成に関するポリシーを設けています）。
- **学習データ・サンプル**: ロイヤリティフリー以外の素材を学習・サンプリング素材として
  取り込まないでください。

## やってはいけないこと（実装で禁止）

| 禁止事項 | 理由 |
|----------|------|
| 既存曲の音源/MIDI を入力にした特徴抽出 | 複製・翻案リスク |
| 既存歌詞のスクレイピング・要約からの作詞 | 翻案リスク |
| 特定アーティストの「声」「作風」のなりすまし | パブリシティ権・不正競争 |
| 既存タイトル/歌詞/商標の流用 | 商標・著作権侵害 |
| 「○○風」と特定楽曲を名指しした生成指示 | 翻案を助長 |

## 推奨ワークフロー

1. トレンドは**ジャンル・テンポ帯・ムード・楽器構成の傾向**といった抽象レベルに留める。
2. 生成プロンプトには「オリジナルであること」「既存曲・実在名を参照しないこと」を明記。
3. リリース前に、AI 生成であることの開示要否を各ストアのポリシーで確認。
4. 疑義がある場合は配信せず、人によるレビューを挟む（半自動運用の利点）。

> 本ドキュメントは技術設計上の配慮指針であり、法的助言ではありません。
> 商用展開の前に、必要に応じて弁護士等の専門家へ相談してください。
